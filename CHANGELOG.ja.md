# 変更履歴

**Nextcloud Sync for Obsidian** の主要な変更点をまとめています。

フォーマットは [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) に準拠し、
バージョニングは [セマンティック バージョニング](https://semver.org/spec/v2.0.0.html) に従います。

掲載は安定版（`x.y.z`）のみです。プレリリースの `-beta.N` ビルドは GitHub の
[Pre-release](https://github.com/siosig/obsidian-nextcloudsync/releases) として公開され、
内容は次の安定版エントリにまとめられます。

> 英語版（原文）は [`CHANGELOG.md`](CHANGELOG.md) を参照してください。

## [0.7.3] - 2026-06-21

- **空ディレクトリの同期が正しく機能するようになりました** — ディレクトリがファイルと対称な第一級エンティティとして管理されます。あるデバイスで作成した空フォルダは、リモートおよびほかのデバイスに伝播されます。あるデバイスでファイルをすべて削除して空になったフォルダは、リモートとほかのデバイスからも削除されます。`2011/` などの年フォルダがファイルを消しても残り続ける長年のバグが修正されました。

## [0.7.2] - 2026-06-21

- **設定画面のツールチップが行のどこをホバーしても表示されるように** — 各行に追加した hover ツールチップが、これまで短い太字のタイトルにしか付いておらず、実際に読む長い説明文をホバーしても何も出ず壊れて見えていました。行全体にラベルを付け、行のどこをホバーしても表示されるようにしました。Server URL の説明には引き続き必要なフル WebDAV エンドポイント（`https://<host>/remote.php/dav/files/<user>/`）を明記し、ホストだけを入力して HTTP 405 で失敗しないように誘導します。サインイン領域は「ブラウザでログイン」を先頭に置き、手動の Username/App password 入力との間に区切りを入れています。
- **コンフリクト領域上限が frontmatter だけでなく本文にも効くように** — 本文のマージ競合も diff3 で領域数を計測するようになり、「最大コンフリクト領域数」を超えた場合に常時自動マージするのではなくコンフリクト失敗ポリシーへ回します。既定（`0`＝無制限）では従来どおり自動マージします。
- **同期ステータスダイアログの時刻を 24 時間表記で統一** — 直近セッションのサマリも各行と同じく `HH:mm`（日跨ぎは日付付き）で表示し、ロケール依存の表記をやめました。

## [0.7.1] - 2026-06-21

- **修正: プラグイン自身のログファイルが同期でエラー/無限再送しないように** — 同期の最中に書き込まれる端末別 debug/sync ログが自分の書き込みと競合し、毎回 "Destination file already exists!" が出ていました。今後はそのログの出力トグルが ON の間だけ同期対象から除外します。
- **frontmatter の競合を黙ってマージせず検出するように** — 3-way マージのバグにより、コンフリクトマーカーを指定していても乖離した YAML frontmatter が黙ってマージされていました。今後は競合した frontmatter を競合解決ポリシーに従って顕在化します。
- **Nextcloud で新規フォルダへの初回アップロードが確実に成功するように** — Nextcloud は親フォルダが未存在のとき標準 WebDAV の 409 ではなく 404 を返します。今後は不足する親フォルダを作成してリトライするため、新端末からの初回同期や深い新規パスへの同期が失敗しなくなりました。
- **サーバへの無駄な往復を削減** — 増分同期用の sync-collection レポートに非対応のサーバでは、一度検出したら以降は毎回試行せず直接フルスキャンに進むようにしました。

## [0.7.0] - 2026-06-20

- **Vault キャッシュ列挙によるモバイル同期の高速化** — ローカルスキャンを、ファイルごとのネイティブファイルシステム呼び出しから Obsidian のメモリ内インデックス（`Vault.getFiles`）読み取りに変更し、モバイルで同期を遅くしていた O(N) のブリッジ往復を排除しました。初回スキャンもチェックサム比較が本当に必要なファイルのみハッシュ計算するようになり、ネットワーク並列度はデスクトップ/モバイル共通（プラットフォーム分岐なし）で端末メモリに応じて自動調整されます。`Vault.getFiles` が取りこぼす dot パスは再列挙して同期対象に保ちます。

## [0.6.1] - 2026-06-20

- **「Reset vault index」保守アクションを追加** — 設定に **Maintenance** セクションを新設し、この端末の同期トラッキングインデックスを初回インストール時の状態に戻すボタンを追加しました（確認ダイアログあり）。次回同期で全件を再スキャンします。Vault やリモートのファイルは一切削除せず、実行中の同期があれば先に中断します。既存の **Last session summary** もこの Maintenance セクションに移動しました。
- **初回同期のプレビュー/承認ステップを廃止** — 初回同期時の「ドライラン」確認モーダルを削除し、初回同期はスキャン後そのまま計画を適用するようになりました。全件再スキャンの承認はリセット操作の確認ダイアログが担います。

## [0.6.0] - 2026-06-20

- **同期が大幅に高速化 — モバイル/Android で止まらなくなりました** — 変更検出を、更新時刻設定（モバイルでは無効）依存から「書き込み直後に観測した stat 署名」での比較に変更し、再同期で Vault 全体を読み直さないように。初回同期の二重ハッシュも解消。転送はメモリ安全なバイト予算付きの並列実行に。ファイルロックは既定 OFF とし `If-Match` 楽観制御で安全性を担保（往復削減・ロストアップデートなし）。大きなサーバー一覧の解析で UI が固まる問題も解消。
- **Sync Status ダイアログをモバイルでも開けるように・Sync 実行単位で表示** — 設定の **Last session summary** ボタンが、デスクトップ／モバイル両方でフル機能の Sync Status ダイアログを開きます。Recent activity は Sync 実行単位でグルーピングされ各区切りに開始時刻を24時間制で表示。ステータスフィルター選択は再起動後も保持されます。

## [0.5.0] - 2026-06-20

- **`.obsidian` 設定フォルダの選択的同期** — 設定の **設定フォルダを同期** を有効にすると、Obsidian 設定フォルダのうち選んだカテゴリ（外観 / テーマ・スニペット / ホットキー / コアプラグイン設定 / ブックマーク）を端末間で同期できます（純正 Sync に倣い各カテゴリ個別トグル）。既定は OFF で、ノートは従来どおり同期されます。**コミュニティプラグイン（`.obsidian/plugins/`）とプラグイン自身の同期状態データベースは一切同期されません**（実行コード・端末固有の状態のため）。設定ファイルの競合は newest-wins（更新時刻が新しい方）で解決し、コンフリクトマーカーで JSON を壊しません。従来の単独設定 **ブックマークを同期** はブックマークカテゴリへ自動移行するため、既存ユーザーは変更なしで動作を維持します。[#1](https://github.com/siosig/obsidian-nextcloudsync/issues/1) を解決。

## [0.4.1] - 2026-06-20

- **ログフォルダが未作成でもログファイルが作られるよう修正** — 指定したログフォルダ（例 `_logs`）がまだ存在しない場合に、端末別の sync／debug ログが黙って一切書き込まれない不具合を修正しました。初回書き込み前に親フォルダを自動作成します。

## [0.4.0] - 2026-06-20

- **同期ステータスダイアログのステータス絞り込み** — ダイアログにステータス別チェックボックス（Uploaded／Downloaded／Deleted／Merged／Conflicted／Local wins／Remote wins／Error）を追加。コンフリクトだけ表示するといった絞り込みができます。
- **ファイルとリモートの比較** — 設定で「リモートと比較（エクスプローラメニュー）」を有効にすると、エクスプローラでファイルを右クリックして、ローカルとリモートの更新日時・チェックサム（一致/不一致バッジ付き）・行 diff を表示するポップアップを開けます。そのまま **push**／**pull** で解決でき、いずれも確認ダイアログを挟みます。
- **ローカルとリモートで内容が異なるファイルが必ず同期されるよう修正** — コンテンツチェックサムを返さないサーバー（ETag フォールバック構成）で、ノートがこの端末とサーバーで異なるのに同期されない不具合を修正。今後は検出して通常のコンフリクト解決で収束させます。
- **端末別ログを .txt 拡張子に変更** — sync／debug ログはプレーンテキストのため、Markdown として描画されて崩れないよう `.txt` 拡張子に変更しました。既存の `.md` ログはそのまま残ります。
- **内部リファクタとドキュメント整合** — 公式 Obsidian 開発者ガイドラインに沿うようコードを整理：vault の監視とエンジン初期化を `onLayoutReady` に遅延、ローカルパスを IO 境界で正規化、モーダルのタイトルをネイティブのタイトルバー表示に統一、diff／エラー表示の配色をテーマ追従に変更。

## [0.3.1] - 2026-06-19

- **メンテナンスリリース（Obsidian 1.11.4 以上が必要）** — 機能変更はありません。0.3.0 の自動レビューで失敗した内部 Lint の問題（説明のない ESLint ディレクティブ）を修正してコミュニティディレクトリへの再掲載に対応し、リポジトリの git フックを再有効化して pre-push ゲートがディレクトリ審査と同じ検査を行うようにしました。

## [0.3.0] - 2026-06-19

- **設定画面を4セクションに再編** — 設定を1本の長いリストではなく **Nextcloud** / **Sync** / **Merge** / **Debug** の見出しごとに整理しました。「Max conflict regions」はスライダー横に数値を表示し、既定値が **0＝無制限**（区間数のサーキットブレーカーが既定でコンフリクトマーカーを強制しなくなりました）になりました。
- **端末別ログ** — 任意のフォルダ（ファジー検索のフォルダ選択 UI。既定は Vault ルート）に出力する2種類のオプトインログ。ファイル名に端末識別子が入るため複数端末で上書きされません。
  - **Sync ログ**（`nextcloud-sync_sync_<端末>.txt`）— 同期ごとに1ブロック追記。ヘッダにプラグインのバージョンと全マージ設定、続けて操作ごとにマーカー・パス・ローカル/リモートのチェックサムとサイズを1行で記録。レベルで*重要イベントのみ*（コンフリクト・マージ・side-wins・エラー）か*全操作*かを選べます。
  - **Debug ログ**（`nextcloud-sync_debug_<端末>.txt`）— 時刻付き診断ログ。詳細度（error / debug / verbose）を選択でき、プラグインのバージョンと全設定値のスナップショットを記録。従来の単一 Vault ルートのデバッグファイルを置き換え、旧「デバッグモード」トグルは自動移行します。
- **コンフリクト件数が減らない不具合を修正** — 一度コンフリクト扱いになったファイルが、その後両側で一致（収束）した場合に次回同期でフラグをクリアするようにし、件数が固着しなくなりました。
- **テレメトリー非収集** — 利用データ・解析・クラッシュレポートは一切収集せず、通信はあなた自身のサーバーとの同期のみです。旧ビルド由来の不要な設定キーも読み込み時に `data.json` から除去します。

## [0.2.10] - 2026-06-17

- **コンフリクト表示の明確化** — 同期ステータスダイアログとサマリーで、**merged**（クリーンな自動マージ完了）と **conflicted**（コンフリクトマーカーがファイルに残存）の2種類を別々にカウント・表示するようになりました。これまでは両者が単一の「conflict」件数にまとめられていたため、対応が必要かどうか判断しにくい状態でした。

## [0.2.9] - 2026-06-16

- **Obsidian の自動ストア審査を通過** — インラインスタイルを CSS クラスへ、UI 文言を sentence case へ修正し、掲載が取り下げられていたコミュニティプラグインの自動チェックをクリアしました。

## [0.2.8] - 2026-06-16

- **より多くの環境でインストール可能に** — 必要な最小 Obsidian バージョンを、実際の最小要件である `1.11.4` に是正しました（従来は事実上、最新版だけを要求していました）。

## [0.2.7] - 2026-06-16

- **手動「Sync now」ボタンと直近24時間のアクティビティ表示** — 同期ステータスダイアログ最上部に **Sync now** ボタンを追加し、スクロール可能な**直近24時間のアクティビティ履歴**（ファイルごとに1行・アイコン先頭のコンパクト表示）を表示。自動同期間隔の変更はリロード不要で即時反映されるようになりました。

## [0.2.6] - 2026-06-11

- **ウォッチモードの自己フィードバックループを修正** — プラグイン自身の書き込み（ダウンロードやマージ結果）をユーザー編集として再検出しなくなり、「ファイル変更時の即時同期」有効時の不要な再アップロード・エラーの連発・意図しないリモート削除を解消しました。
- **1 ファイルの失敗で同期全体が中断しなくなりました** — ファイル単位のエラー（サーバー側 403 など）は記録して残りのファイルの同期を継続します。ネットワークエラーは従来どおりバックオフ付きで再試行キューに積まれます。
- **同期ステータスダイアログにエラー詳細を表示** — ステータスバーをクリックすると、エラー件数だけでなく「前回同期のエラー」一覧（失敗ファイルごとに 1 行、クリックでファイルを開く）が表示されるようになりました。
- **状態データベースの保存を直列化** — ウォッチモードの並行同期が競合して同期状態ファイルが破損・消失する問題を解消しました。

## [0.2.5] - 2026-06-11

- **リモート削除のスコープガード（セキュリティ強化）** — サーバーが報告する削除を同期スコープと照合してから適用するようにし、悪意ある／侵害されたサーバーが同期フォルダ外（`.obsidian/` 配下など）のファイルをクライアントに削除させられないようにしました。

## [0.2.4] - 2026-06-10

- **ダウンロードの堅牢化** — アトミック書き込みの前に親フォルダを作成し、ファイル単位の非ネットワークエラーで残りファイルの同期が止まらないようにしました。

## [0.2.2] - 2026-06-09

- **衝突解決ポリシー** — クリーンに自動マージできない衝突時の挙動を選択可能に: 両側を一切触らず次回再試行（**エラー**＝既定）／**ローカル優先**／**リモート優先**／**コンフリクトマーカー**。設定画面で選択できます。
- **マージ対象をテキストファイルに限定** — テキストマージの対象は設定可能な拡張子（既定 `md`・`txt`）のみ。それ以外（画像・PDF・バイナリ等）はマージせず、コンフリクトマーカーでファイルを破壊することがなくなりました。衝突時の既定は `<<<<<<<` マーカー埋め込みから **エラー**（スキップして次回同期で再試行）に変更されています。

## [0.2.0] - 2026-06-09

Nextcloud 特化同期エンジンの初回公開リリース（0.2.0 〜 0.2.1）:

- **端末間の削除が確実に伝播** — ある端末で削除したファイルが次回同期で復活する問題を修正（sync-token 処理の修正。削除は内容検証付き・ゴミ箱経由で復元可能、大量削除のサーフィン防止ガード付き）。
- **自動マージの結果がサーバーに反映** — マージ結果をアップロードして全端末が収束。同じコンフリクトが毎回再発しなくなりました。
- **モバイル診断** — Debug ログがモバイルでも端末別 `nextcloud-sync_debug_<端末>.txt` を出力し、Sync now が結果通知を表示。
- **モバイル（iOS / Android）対応** — モバイルでも動作するようになりました。プラットフォーム対応の分岐によりデスクトップの挙動は不変です。
- **クリック可能なステータスバー → 同期ステータスダイアログ** — ステータスバーをクリックすると、現在の同期状態・前回同期・未解決コンフリクト（0.2.6 以降はファイル単位のエラー詳細も）をまとめたダイアログが開きます。
- **「Sync now」を設定画面の最上部へ移動**し、認証済みのときのみ有効化。サインインすればすぐに同期を実行できます。
- **プラットフォーム対応のデフォルト設定** — 通信並列数・起動時同期・最大ファイルサイズなどの既定値をプラットフォームごとに最適化。
- **YAML フロントマターも自動マージ** — 重複しないフロントマターの編集は diff3 でクリーンに統合し、同一行を双方が変更した場合のみコンフリクトマーカーにフォールバックします。
- **Dry-run でのコンフリクト結果の明確化** — 初回同期プレビューがコンフリクト解決の結果を説明し、各コンフリクトファイルをクリックするとマージ後の内容（変更前後）をプレビューできます。
- **汎用 WebDAV より高速な同期** — 内容ハッシュと Nextcloud の `sync-token` を突き合わせ、毎回リモートツリー全体を再帰的に走査するのではなく、実際に変更された分だけを転送します。更新日時ベースの WebDAV プラグインより同期が明確に速くなります。
