# 変更履歴

**Nextcloud Sync for Obsidian** の主要な変更点をまとめています。

フォーマットは [Keep a Changelog](https://keepachangelog.com/en/1.1.0/) に準拠し、
バージョニングは [セマンティック バージョニング](https://semver.org/spec/v2.0.0.html) に従います。

掲載は安定版（`x.y.z`）のみです。プレリリースの `-beta.N` ビルドは GitHub の
[Pre-release](https://github.com/siosig/obsidian-nextcloudsync/releases) として公開され、
内容は次の安定版エントリにまとめられます。

> 英語版（原文）は [`CHANGELOG.md`](CHANGELOG.md) を参照してください。

## [0.7.28] - 2026-07-08

### Fixed
- 前バージョンの自動レビューが失敗し、コミュニティプラグインディレクトリから外れていた不具合を修正。Obsidian の審査は UI 文言の `sentence-case` ルールを無効化することを許可しておらず、リテラルなフォルダ名 `.git` に対する誤検知を回避するためルール抑制コメントを使っていたのが原因でした。抑制ではなく実際の原因を修正: ルールのブランド一覧が "git" をどこに出現しても "Git" に強制変換していましたが、これは常に小文字が正しい実在のファイルシステムのフォルダ名に対しては誤りだったため、ブランド一覧の上書きから "Git" を除外しました。ユーザー向けの挙動変更はありません。

## [0.7.27] - 2026-07-07

### Fixed
- `.git` と `.trash` フォルダを同期しないように。Obsidian のローカルゴミ箱フォルダ（`.trash`）とリポジトリの `.git` フォルダが、これまで全デバイスへアップロード／ダウンロードされていました。`.trash` の同期は各デバイスのゴミ箱を散らかし、プラグイン自身の「ゴミ箱へ移動して削除」挙動と競合して churn を起こします。`.git` をファイル単位で同期するとリポジトリが破損し得ます。両者を *Excluded folders* リストに関わらず**常に除外**するようにしました（設定画面が以前から「除外済み」と説明していた挙動に実装を合わせた形です）。これは対象を絞った除外で、vault ルート直下のそれ以外の dot フォルダ（例: `.archive/`）は従来どおり同期します。既にサーバーへ上がっている `.trash`/`.git` は削除せずそのまま残します（不要ならサーバー側で手動削除してください）。

## [0.7.26] - 2026-07-06

### Fixed
- 「A sync is already in progress」で永久に詰まる不具合を修正。最初のステップ（サーバー接続と capabilities 取得）が失敗するとエンジンが「同期実行中」フラグを立てたままエラーを握り潰し、以降すべての同期が弾かれ、起動時同期が同じ失敗を再誘発するため再起動しても直りませんでした。この接続ステップをエラーハンドリングの内側へ移し、失敗が表面化して次回試行で回復するようにしました。
- **Network timeout** 設定が実際のリクエストに適用されるように。従来は一切配線されておらず、接続だけ受け付けて応答しないサーバーで同期が無限にハングし得ました（上記ロックも誘発）。全 WebDAV リクエストが設定タイムアウト（既定 30 秒・0=無制限）で上限付けられ、詰まったリクエストは失敗して次回同期でリトライします。
- 全ソース監査で、1 ステップの失敗が内容を無音で喪失／復活させ得るデータ安全性経路を一括修正：クリーンマージの upload 失敗時もフラグ維持／リモート削除が成功したときだけ追跡解除／中断されたアトミック書き込みで新旧両方を失わない／大容量（チャンク）アップロードも小ファイルと同じ楽観的排他を守る／同期中クラッシュ後はディスク状態から復旧／空 base マージが2独立編集を1行に融合しない。加えて force-resolve / restore の二重クリックガード、モバイル watch モードの実行時無効化。

## [0.7.25] - 2026-07-06

### Fixed
- クライアント側キャッシュに起因する同期の恒久的な詰まりを修正しました。アプリのネットワークキャッシュが、あるファイルの不完全な異常レスポンスを一度でも保持すると、そのファイルは再起動や「Sync now」「Use remote」を何度試しても永久に同期に失敗し続けることがありました。全リクエストでこのキャッシュを明示的に無効化したため、異常なレスポンスが再利用されることはなくなり、次回同期では必ずサーバーへ新規問い合わせして正しい結果を得られます。

### Changed
- トラブルシューティング用ログを「見つかる 1 ファイル」に再設計しました。**Debug → Enable logging** を ON にすると、端末ごとに単一のファイル `nextcloud-debug_<device>.txt` を Vault ルートに出力します（従来の 2 ファイルを統合）。ON にした瞬間に書き込み先のファイル名を明示する通知が出て、Obsidian は **設定 → ファイルとリンク → すべてのファイル拡張子を認識** が ON でないと `.txt` を隠すことも案内します（OS や Nextcloud 上でも開けます）。ログの書き込みに失敗した場合も無言ではなく通知が出るようになりました。

## [0.7.24] - 2026-07-05

### Added
- フロントマターが本文から独立した競合戦略を持つようになりました。`.md` ファイルのフロントマターブロックと本文を別々に解決し、新設の **Frontmatter strategy** 設定（既定: merge）がフロントマターを、既存のファイル種別マージ戦略が本文を担当します。
- 2段階の競合解決：一次戦略（frontmatter / auto-merge / other-file）だけでは衝突を完全に解決できない場合、残った衝突領域・フィールドは新設の明示的な **Conflict strategy**（既定: conflict markers）にフォールバックし、all-or-nothing の判定ではなくなりました。
- **Mirror from remote** がクリック直後に進捗ダイアログを開き、各段階（リモート読み込み・比較、続いてダウンロード/削除）をリアルタイム表示するようになり、固まって見える問題を解消しました。
- mass-delete 安全上限を新設の **Advanced (use with caution)** セクションで設定可能にしました（`-1` = 自動 / `0` = 無制限 / 固定値）。

## [0.7.23] - 2026-07-04

### Fixed
- モバイルの **Sync interval** スライダーが実態どおりの表示になりました。モバイルでは OS が背景タイマーを止めるため定期同期はもともと発火しませんが、スライダーはグレーアウト状態でもデスクトップ既定の 15 分を表示していて、「15分ごとに同期している」ように見えていました。モバイルの新規インストールでは初期値を 0（手動のみ）にし、無効スライダーの表示が実挙動と一致するようにしました。デスクトップは変更なし。既に設定済みの間隔値もそのまま保持します。

## [0.7.22] - 2026-07-04

### Added
- ウォッチモードでフォルダの変更も即時伝播するようになりました。**ファイル変更時の同期** を ON にしていると、フォルダの作成・削除・リネームがサーバーへ即時反映されます（作成→フォルダを作成、削除→ゴミ箱へ、リネーム→移動）。従来ファイルだけだった即時伝播がフォルダにも揃います。伝播中はステータスバーに表示され、設定の説明文も「対象はファイル**とフォルダ**、作成・編集・削除・リネームすべて（デスクトップ限定）」と実態どおりに修正しました。
- **設定 → Maintenance** に新しい **「リモートの状態に合わせる（Mirror from remote）」** アクションを追加しました。この端末の Vault をリモートの現在状態に完全に一致させます。リモートにあるものをすべてダウンロードし、リモートに無くなったローカルのファイル／フォルダを Obsidian のゴミ箱へ移動します（復元可能）。実行前に「ダウンロード予定数／削除予定数」を確認ダイアログで提示し、ローカルの未同期の変更は破棄されます。大量削除の安全上限はバイパスします（あなたが明示的にリモートを正と宣言したため）が、リモート一覧を完全に取得できない場合は何も削除せず中止します。他の同期ツールから移行した端末をリモートに従わせたいケースに便利です。

## [0.7.21] - 2026-07-04

### 追加
- Sync status ダイアログに一括強制解決を追加しました。ドロップダウンを1件ずつ操作する代わりに、戦略（Use remote / Use local / Latest modified / Biggest size）を一度選ぶだけでリスト中の全競合にまとめて適用できます。同値（更新時刻・サイズが等しい）は手を付けず、失敗したものはコンフリクトのまま残すので、黙って失われることはありません。

### 修正
- 競合時の frontmatter（YAML プロパティ）を、テキストではなく構造化データとして解決するようにしました。ノートの `---` ブロックの中に競合マーカー（`<<<<<<<` / `>>>>>>>`）が入り込むこと（Obsidian のプロパティが壊れ、次の同期でさらに入れ子になる原因）がなくなりました。リスト項目（tags・aliases・related）は本物の 3-way 集合マージになり、片方の端末で削除したタグが復活せずきちんと消え、表記ゆれ（`#tag` と `tag`）は1つにまとまり、サーバー側ツールによる外部からの変更も正しく伝播します。
- 「Use remote」「Use local」「Latest modified」「Biggest size」の強制解決が本物のマーカーなし版を復元するようになりました。テキスト競合でノートにマージマーカーが書き込まれた場合、これまで Sync status ダイアログからの強制解決はマーカー入りの内容を再同期するだけで警告だけが消えていました。今回から競合時に両方の clean 版をスナップショットし、ノートが実際に綺麗になったときにだけ競合を解除します。

## [0.7.20] - 2026-07-02

### 修正
- コンフリクトマーカーの消し忘れ（手動解決で残った `<<<<<<< LOCAL` または `>>>>>>> REMOTE` 単体の行）が原因のデッドロックを解消しました。従来この状態のファイルは push されないまま保留され、毎回の同期でコンフリクトに戻り続けていました。今後は通常のマージを通って収束し、ローカル・リモート双方から余分なマーカーを取り除きます（self-heal）。完全なマーカー集合や入れ子マーカーは従来どおり保留します。

### 追加
- Sync status ダイアログに per-file の強制解決を追加しました。各コンフリクトファイルにドロップダウン（Use remote / Use local / Latest modified / Biggest size）と Apply ボタンが付き、その場で即座に解決できます。同値（更新時刻・サイズが等しい）の場合は何もせず、失敗時はコンフリクト状態を維持します。

## [0.7.19] - 2026-07-01

### 変更
- フロントマターのマージがセマンティックになりました。`tags`・`aliases`・`related` などの配列フィールドは端末をまたいでユニオンマージ（重複排除）され、`title`・`status` などのスカラーフィールドは 3-way 比較で解決されます。片方だけの変更は自動解決し、真のコンフリクトは新しい設定ポリシー（最終更新優先 / リモート優先 / ローカル優先。既定: 最終更新優先）で決定します。フロントマターのないノートは影響を受けません。設定は「Conflict Resolution」セクションの「Frontmatter scalar conflict [experimental]」に追加されています。

## [0.7.18] - 2026-06-30

### 修正
- コンフリクトのマージ精度を向上しました。同じノートを 2 台の端末で編集した際、最後に同期した版を本当の 3-way マージのベースとして使うようになり、自動マージがよりきれいになり編集の取りこぼしも減りました。
- 既存のコンフリクトマーカーが同期のたびに再マージされ、該当ノートが際限なく肥大化していた再入ループを停止しました。
- no-op の「引き分け」がルート ETag の高速経路で必要な再同期を短絡してしまう不具合を修正し、変更を取りこぼす経路を塞ぎました。

## [0.7.17] - 2026-06-30

### 修正
- スマートフォンで、0.7.16 で各設定スライダーの隣に追加した編集可能な数値ボックスが行いっぱいに広がり、スライダーを画面外へ押し出していました。数値ボックスをコンパクトな幅に固定し、数値ボックスとスライダーの両方が見えて操作できるようにしました。

## [0.7.16] - 2026-06-30

### 変更
- コンフリクト解決をファイル種別ごとに選べるようになりました。旧来の3設定（自動マージのトグル・frontmatter コンフリクト戦略・マージ失敗時ポリシー）を、**自動マージファイルの拡張子リスト**＋**自動マージファイルの戦略**＋**その他ファイルの戦略**に集約しました。各戦略は Merge（3-way）/ Latest modified / Biggest size / Local wins / Remote wins のいずれかで、コンフリクトは必ず決着します（保留・エラーのモードはありません）。サイズ/更新時刻が同値の場合は両側不変で次回同期に再評価します。旧設定は自動移行され、不要なキーは除去されます。
- Merge 戦略では、非テキスト（バイナリ）ファイルにコンフリクトマーカーを書き込む代わりに一切触らずフラグのみ立てるようになりました。また、内容を重複させてしまうマージ（空 base 由来のまれな reconcile 不具合）を、破損結果を書かずコンフリクト扱いへ降格する膨張ガードを追加しました。

### 追加
- 各数値設定のスライダーの隣に編集可能な数値入力ボックスを追加し、キーボードで正確な値を入力できるようにしました。範囲外の値はクランプ、不正な入力は直前の値へ復帰し、入力ボックスとスライダーは常に同期します。

## [0.7.15] - 2026-06-29

### 変更
- 「最大ファイルサイズ」設定がアップロードだけでなくダウンロードにも適用されるようになりました。上限を超えるリモートファイルは取得前にスキップされます（サーバが PROPFIND の `getcontentlength` で申告するサイズを正典として判定）。これにより巨大なノートがモバイルでメモリに読み込まれてアプリをクラッシュさせることがなくなります（issue #8）。スキップは非破壊的かつ自己修復的で、上限を引き上げれば次回の同期で通常どおりダウンロードされます。

## [0.7.14] - 2026-06-27

### 修正

## [0.7.13] - 2026-06-27

### 変更
- **起動時同期の操作を簡素化＋スライダー範囲を整理**：独立していた *Sync on startup* トグルを *Startup sync delay* スライダー 1 本に統合しました（`0`＝起動時同期しない、`1`〜`10` 秒＝遅延秒数）。数値スライダーの範囲・刻みも見直し、*Sync interval* は 4 分刻み（0〜60、0＝手動のみ）、*Network concurrency* は 0〜60 の 4 刻み（0 は 1 として動作）になりました。

### マイグレーション

## [0.7.12] - 2026-06-27

### 変更
- **利用価値の少ない 5 設定を UI から撤去し固定値化**：ファイルロックは常に OFF（常時有効の `If-Match` プリコンディションが LOCK/UNLOCK の往復なしでロストアップデートを防止）、チャンクアップロードは常に ON、チャンク閾値はプラットフォーム別の固定値（デスクトップ 50 MB / モバイル 20 MB）、最大コンフリクト領域数は常に無制限、*Compare with remote* はエクスプローラメニューとコマンドで常時利用可能になりました。

### マイグレーション
- これら 5 設定に以前保存していたカスタム値は、アップグレード時に自動でクリーンアップされ、全インストールが同一の単一経路に揃います。

## [0.7.11] - 2026-06-26

### 変更
- **Debug 設定を単一トグルに縮小**：Device name と Log folder の入力欄を撤去し、*Enable logging (troubleshooting)* のみを残しました。デバイス名は自動導出され、ログは常に vault root に出力されます（sync ログは全操作を記録、debug ログは verbose）。

### マイグレーション
- 以前に設定したカスタムのデバイス名・ログフォルダは、アップグレード時に自動／vault root の既定値へリセットされ、全インストールが同一経路に揃います。過去のカスタムフォルダに既に書かれたログファイルはそのまま残します。

### 変更

## [0.7.9] - 2026-06-25

- **修正: 非常に長い名前のノートが Android に同期されない問題** — ファイル名自体はファイルシステムの「1 コンポーネント 255 バイト」上限内なのに、アトミック書き込みで使う一時ファイルが 18 バイトの接尾辞を付けることで一時ファイル名が上限を超え、`FILE_NOTCREATED` エラーでダウンロードに失敗していました（日本語は 1 文字 3 バイトのため、タイトルが約 80 文字で発生）。一時ファイルを「対象と同じディレクトリ内の短い固定長ハッシュ名」に変更し、その長さが最終ファイル名に依存しないようにしたため、ファイル名自体が上限内なら確実に書き込めます（rename もアトミックを維持）。OS 的に回避不能な「名前そのものが 255 バイト超」のケースは、不透明なエラーではなく「ファイル名が長すぎます（N バイト / 上限 255）」という分かりやすいメッセージで表示します。

## [0.7.8] - 2026-06-23

- **修正: モバイルでダウンロードが拒否されるリグレッション** — 0.7.7 で追加したダウンロード安全ガードが、受信 `arrayBuffer.byteLength` とサーバ申告 `getcontentlength` の不一致を一律に拒否していた。iOS では Obsidian の `requestUrl` が返すバイト長がサーバの content-length と正当に食い違う（実サーバで検証: PROPFIND==GET Content-Length==実バイトは整合、クライアント側の数値のみマルチバイトで差が拡大）ため、正常なダウンロードが拒否され remote→local 同期が停止していた。ガードを「非空申告に対し本文が完全な空(0 バイト)の場合のみ拒否」に限定。書き込み後検証（書いた長さの自己照合）は影響なし。
- **修正: モバイルでの Sync Status フィルタの可読性** — ステータス別フィルタを、詰まって重なるチェックボックスから、選択状態が明確なフル幅チップ表示に変更。
