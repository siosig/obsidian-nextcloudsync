# Nextcloud Sync for Obsidian

**複数のデスクトップ・モバイル端末でノートを書いている方へ。**

あなたが払うコストは、初回インストール時に Vault 内の全ファイルをインデックス化するための待ち時間だけです。それだけで、今日から次の恩恵を得られます：

- **書いた内容が失われない** — マージロジックにより、複数端末で同時に編集しても内容が消えることなく、Vault が常に整合した状態に保たれます。
- **同期時間を意識しなくなる** — Nextcloud の高速な差分同期により、同期処理を気にすることなく作業に集中できます。
- **Nextcloud 以外でも動作** — 機能は一部限定されますが、汎用 WebDAV サーバーでも同様に利用できます。

---

Obsidian Vault と Nextcloud を双方向同期するプラグインです。汎用的な WebDAV ではなく、**Nextcloud 専用**に作られています。

多くの「WebDAV 同期」プラグインはサーバーを単なるファイル置き場として扱い、更新日時を比較してファイルをコピーするだけです。**Nextcloud Sync** は Nextcloud 固有の API（Capabilities、ファイル ID、チェックサム、バージョン、ロック、Login Flow v2）を活用し、同期を *安全* かつ *高速* かつ *簡単* にします。同時に、それらの API が使えない場合は標準 WebDAV へ自動的にフォールバックします。

## なぜ Nextcloud 特化なのか（汎用 WebDAV との違い）

| 観点 | 汎用 WebDAV プラグイン | **Nextcloud Sync** |
|------|----------------------|--------------------|
| 変更検出 | 更新日時（時刻ずれに弱く、誤った再アップロードが発生） | **内容ハッシュ** ＋ Nextcloud の `sync-token` / checksums capability による真の差分同期 |
| リネーム / 移動 | 削除＋再作成（履歴を失い、全デバイスで再ダウンロード） | **ファイル ID（`OC-FileId`）追跡** — リネームは全デバイスでリネームのまま、履歴を保持 |
| 削除の安全性 | 完全削除 | **Nextcloud ゴミ箱** 経由 — 復元可能で、不可逆な `DELETE` を行わない |
| セットアップ | アプリパスワードを手動でコピー＆ペースト | **Login Flow v2** — ブラウザで承認するだけで認証情報を自動発行・保存 |
| 大容量ファイル | スキップ、または単一 `PUT` で失敗 | **チャンクアップロード** — 分割・再開可能・チェックサム検証付き |
| 同時編集 | 他者の書き込みを祈るのみ | **楽観的同時実行** — 全更新に `If-Match` を付与し、他デバイスが変更した場合はコンフリクトに変換（ロストアップデート防止）。ロック往復なし。サーバー側 **Files Locking** は任意で併用可 |
| 誤操作からの回復 | なし（手元のコピーが全て） | **サーバーバージョン履歴** — Obsidian 内から過去版を一覧・復元 |
| サーバー停止時 | 不可解なエラー / 中途半端な書き込み | **メンテナンスモード検出**（`/status.php`）と Nextcloud エラーの整形表示 |
| 機能の認識 | なし | **Capabilities 探索**（`/ocs/.../capabilities`）— サーバーが対応する機能だけが有効化される（Progressive Enhancement） |

非 Nextcloud の WebDAV サーバーに接続した場合は、Nextcloud 専用機能を自動的に無効化し、標準的な再帰 WebDAV 同期へフォールバックします。機能は減りますが、そのまま動作します。

## 機能

### コア同期
- Obsidian と Nextcloud の**双方向同期**。
- **ハッシュベース差分同期** — 変更されたファイルのみ転送（全件再スキャンなし）。1,000 ファイル規模の Vault でも数秒で収束。
- **アトミック書き込み** — ダウンロード中に転送が中断されても、書きかけや 0 バイトのファイルを Vault に残さない。
- Nextcloud ファイル ID による**リネーム / 移動追跡** — ノートを移動しても全デバイスで再アップロードしない。
- **ゴミ箱経由の削除** — リモート削除は Nextcloud のゴミ箱へ（復元可能）。削除をローカル Vault に反映する際は、削除手段を固定せず **Obsidian の「削除されたファイル」設定**（システムのゴミ箱／`.trash` へ移動／完全削除）に従います。フォルダや、Vault が追跡しないファイル（設定フォルダ配下のファイル等）も適切に削除します。
- **Vault ごとの独立設定** — Vault ごとに別の Nextcloud サーバー / アカウントを設定でき、状態が混在しない。
- **定期自動同期**（間隔は設定可能、`0` で手動のみ）と **Sync now** コマンド。
- **ファイル変更時の即時同期（ウォッチモード）** — ローカルファイルを編集すると自動で同期（編集が止まって約2秒後）。設定でトグル ON/OFF、定期同期と併用可。
- **堅牢なリトライ** — 失敗ファイルはスキップ＆キューに積み、次回同期で指数バックオフ付き再試行。Wi-Fi 切断後も自動再開。
- **標準 WebDAV フォールバック** — 任意の WebDAV サーバー（再帰）で動作し、Nextcloud 機能は自動無効化。
- **同期ステータスダイアログのステータス絞り込み** — ダイアログ上部にステータス別チェックボックス（Uploaded／Downloaded／Deleted／Merged／Conflicted／Local wins／Remote wins／Error）を用意。コンフリクトだけ表示するといった絞り込みが可能です。既定は全 ON、選択は保存され Obsidian の再起動を跨いで永続し、すべてのセクションに作用します。
- **ファイルとリモートの比較**（オプトイン）— 設定で「リモートと比較（エクスプローラメニュー）」を有効にすると、エクスプローラでファイルを右クリックして、ローカルとリモートの更新日時・チェックサム（一致/不一致バッジ付き）・行 diff を表示するポップアップを開けます。そのまま **push**（ローカルでリモートを上書き）／**pull**（リモートでローカルを上書き）で解決でき、いずれも確認ダイアログを挟みます。トグルは再起動不要で即時反映されます。
- **端末別ログ** — 任意のフォルダ（ファジー検索のフォルダ選択 UI。既定は Vault ルート）に出力する2種類のオプトインログ。ファイル名に端末識別子が入るため複数端末で上書きされません。
  - **Sync ログ**（`nextcloud-sync_sync_<端末>.txt`）— 同期ごとに1ブロック追記。ヘッダにプラグインのバージョンと全マージ設定、続けて操作ごとにマーカー・パス・ローカル/リモートのチェックサムとサイズを1行で記録。レベルで*重要イベントのみ*（コンフリクト・マージ・side-wins・エラー）か*全操作*かを選べます。
  - **Debug ログ**（`nextcloud-sync_debug_<端末>.txt`）— 時刻付き診断ログ。詳細度（error / debug / verbose）を選択でき、プラグインのバージョンと全設定値のスナップショットを記録。コンソールのないモバイルでの不具合調査に有用。調査が済んだら OFF にしてファイルを削除してください。
- **Vault インデックスのリセット**（設定 → Maintenance）— この端末の同期トラッキングインデックスを初回インストール時の状態に戻します（確認ダイアログあり）。次回同期で全件を再スキャンします。Vault やリモートのファイルは一切削除しません。同期状態が不整合になったときの復旧に使えます。

### コンフリクト安全性（内容を消さない）
- **自動マージ**（`reconcile-text` / diff3）— 異なる領域の編集を統合（両者が重複しない行を変更していれば YAML フロントマターも統合）。既定 ON。
- **拡張子によるマージ対象の限定** — テキストマージの対象は設定可能な拡張子（既定 `md`・`txt`）のみ。それ以外（画像・PDF・バイナリ等）はマージせず、マーカーも書き込まない。
- **衝突解決ポリシー** — クリーンにマージできない場合の挙動を選択: **エラー**（両側を一切触らず、エラーとして報告し次回同期で再試行＝既定）／**ローカル優先**（ローカルでリモートを上書き）／**リモート優先**（リモートでローカルを上書き）／**コンフリクトマーカー**（Git 形式 `<<<<<<< LOCAL` / `=======` / `>>>>>>> REMOTE` をファイルに書き込み。テキストファイルのみ。それ以外はエラーにフォールバック）。
- ステータスバーの**コンフリクトバッジ** — 未解決件数を表示し、0 で通常表示に戻る（`#conflict` タグ検索と併用可）。

### Nextcloud 強化機能
- **Login Flow v2** — アプリパスワードを手動発行・貼り付けする代わりに、ブラウザ承認だけでセットアップ。認証情報は Obsidian の credentials ストアに保存され、`data.json` に**平文保存しない**。
- **サーバーバージョン履歴** — アクティブなノートについてサーバー保持の全版を新しい順に一覧し、確認のうえ任意の版をアトミックに復元。復元後の内容は無限コンフリクトを起こさずクリーンに同期される。
- **チャンクアップロード** — チャンク閾値を超える大容量ファイル（画像・PDF・音声）を分割・再開可能にアップロード。中断時に不完全なファイルを公開せず、完了時にチェックサム検証。メモリ保護のため別途絶対上限 `maxFileSizeMB` を設定。
- **Files Locking**（*実験的・既定 OFF*）— 任意で、各更新の直前にファイル単位のサーバーロックを取得し直後に解放します。**既定 OFF**：ロストアップデート対策は常時付与の `If-Match` プリコンディション（他クライアントが変更済みなら 412 が返り、エンジンがコンフリクトに変換）で担保し、ファイルごとの LOCK/UNLOCK 往復を省きます。二重の安全策が欲しい場合に有効化してください。Nextcloud のファイルロックアプリが必要で、アプリが無い場合は無効のままです。

## モバイル（iOS / Android）

モバイルに対応しています。OS 制約に合わせた差分があります（デスクトップの挙動は不変）:

- **モバイルでは自動同期が既定 OFF**。OS がバックグラウンドのタイマーを止めるため、定期自動同期・編集時同期（watch）は無効（グレーアウト）。**Sync now**（手動）か、**起動時同期**（モバイル既定 OFF）を有効化して起動数秒後に1回同期します。
- **起動時同期**は両プラットフォーム共通の新設定（デスクトップ既定 ON・1 秒／モバイル既定 OFF）。
- **大容量ファイルはモバイルでスキップ**（「最大ファイルサイズ」上限超。`0` で無制限）。メモリ不足によるクラッシュを回避。スキップは通知します。
- **モバイルは進捗表示なし** — 通知はエラー時のみ。
- **通信並列数**を設定可能。初回の既定値は端末の搭載メモリから導出します（デスクトップ・モバイル共通で、プラットフォームによる分岐はありません）: RAM 8 GB 以上で **16**、4 GB 以上で **8**、それ未満で **4**、端末がメモリ量を報告しない場合（モバイルに多い）は **3**。転送は並列数と「同時転送バイト総量」の両方で上限化（モバイルは小さめ）され、大きなファイルでもメモリを使い切りません。さらに同一フォルダへのアップロードは直列化し、サーバー側ロック競合を回避します。
- **Wi-Fi のみ同期**でセルラー時にスキップ（Android/デスクトップ）。**iOS は非対応**（種別検出 API が無く、トグルはグレーアウト）。
- **モバイルでは Sync now が結果通知を表示**（アップロード / ダウンロード / コンフリクト件数、または「最新です」）。ステータスバーが無いため。未認証時はグレーアウトし、設定画面に赤帯の「未ログイン」警告を表示。
- デバッグモード（診断ログ）はモバイルでも利用可能で、同期挙動は変えません。

## 動作要件

- **Obsidian** `1.11.4` 以上（本プラグインは `1.11.4` で追加された secret-storage API を使用します）。デスクトップ（Electron）とモバイル（iOS / Android）に対応。
- Nextcloud 固有機能には **Nextcloud** Hub 26 「Winter」（サーバ `33`）以上を**推奨**。これより古い Nextcloud サーバーも接続拒否はせず同期可能ですが、設定画面に推奨アップグレードの警告が表示され、一部機能が制限される場合があります。標準 WebDAV サーバーではコア同期にフォールバック。
- Nextcloud アカウント。認証は **Login Flow v2**（推奨）または手動発行の**アプリパスワード**（メインパスワードは使用不可）。

## インストール

### コミュニティプラグインブラウザから（推奨）
1. Obsidian で **設定 → コミュニティプラグイン** を開く。
2. 制限モードを解除し、**閲覧** をクリックして **Nextcloud Sync** を検索。
3. **インストール** し、**有効化**。

### 手動インストール
1. 最新の [GitHub Release](../../releases) から `main.js` と `manifest.json`（あれば `styles.css`）をダウンロード。
2. `<YourVault>/.obsidian/plugins/nextcloud-sync/` にコピー。
3. Obsidian をリロードし、**設定 → コミュニティプラグイン** で **Nextcloud Sync** を有効化。

## はじめかた

1. **設定 → Nextcloud Sync** を開く。
2. **サーバー URL** を入力（例: `https://cloud.example.com`）。
3. 認証する:
   - **推奨:** **ブラウザでログイン**（Login Flow v2）をクリックし、ブラウザで承認すると認証情報が自動入力・保存される。または
   - **ユーザー名** と手動発行の**アプリパスワード**を入力。
4. （任意）自動同期間隔・**ファイル変更時の即時同期**（ウォッチモード）・自動マージ・チャンク閾値・ロックオプションを調整。
5. **Sync now** コマンドを実行（または定期同期を待つ）。初回は Vault とリモートを全件スキャンして必要なものを転送し、以降は差分同期になります。

Vault は Nextcloud 側で Vault 名のフォルダ配下に同期され、複数 Vault がきれいに分離されます。

## Nextcloud サーバー側機能の有効化

2 つの強化機能はサーバー側の Nextcloud アプリに依存します。いずれも **Nextcloud 管理者が一度だけ有効化**すれば十分です。プラグインは capabilities API で検出し、アプリが無ければ該当機能を自動的に無効化します（エラーにはなりません）。

### File Locking

ロックは Nextcloud の **Temporary files lock** アプリ（アプリ ID `files_lock`、Nextcloud 24 以降。v34 以降は Hub に同梱）を使用します。

- **Web UI（管理者）**: 管理者でログイン → 右上のプロフィールメニュー → **Apps** → 「**Temporary files lock**」を検索 → **有効化**（未インストールなら先にダウンロード）。
- **コマンドライン（`occ`）** — Web サーバーユーザー（多くは `www-data`）で実行:
  ```bash
  sudo -u www-data php /var/www/nextcloud/occ app:enable files_lock
  ```
- その後、プラグイン設定の **File Locking (Experimental)** を ON にして再同期してください。（Docker の場合: `docker exec -u www-data <container> php occ app:enable files_lock`）

### バージョン履歴

サーバー側のバージョンは組み込みの **Versions** アプリ（アプリ ID `files_versions`）が提供します。標準インストールでは**既定で有効**のため通常は作業不要です。

- 無効化されている場合のみ再有効化: **Apps → Versions → 有効化**、または:
  ```bash
  sudo -u www-data php /var/www/nextcloud/occ app:enable files_versions
  ```
- バージョンはファイル変更時に自動作成され、Obsidian の **Show version history** コマンドで閲覧・復元できます。
- 過去版が存在するには、サーバー上で当該ファイルが一度以上変更されている必要があります。保持世代数は管理者が `config.php` の `versions_retention_obligation` で設定します。

### その他の確認しておきたい設定

必須ではありませんが、セルフホスト環境では安定した同期のために確認が必要になりがちな項目です。いずれもサーバー側（管理者）の設定です。

- **信頼済みドメイン（trusted_domains）** — 接続先ホストが `config.php` の `trusted_domains` に登録されている必要があります。未登録だとサーバーがリクエストを拒否します。必要に応じてドメイン／IP を追加してください。
- **HTTPS とリバースプロキシ（Login Flow v2 で重要）** — リバースプロキシ配下では `overwriteprotocol => 'https'`、`overwritehost`、`overwrite.cli.url`、`trusted_proxies` を正しく設定してください。これらが誤っていると、Login Flow v2 が返す URL（やダウンロード URL）のスキーム／ホストが不正になり失敗します。プラグインのサーバー URL は必ず `https://` を使用してください。
- **アップロードサイズ上限（チャンクアップロード／大容量添付向け）** — PHP の `upload_max_filesize`・`post_max_size`、および Web サーバーのボディ上限（nginx の `client_max_body_size`、例: `0` か大きい値）を引き上げてください。チャンクアップロードは小さく分割しますが、最終結合や非常に大きなファイルではこれらの上限に達します。
- **リクエストタイムアウト** — 大規模 Vault や大きいファイルでは、PHP の `max_execution_time` や php-fpm／Web サーバーのタイムアウト（例: nginx の `fastcgi_read_timeout`）を延長してください。プラグイン側の通信タイムアウトは設定画面で調整できます。
- **ブルートフォース保護** — Nextcloud は同一 IP からの連続リクエストを抑制し、特に複数デバイスが同一ネットワークから同期する場合や認証エラー後に HTTP 429 を返すことがあります。発生する場合は **管理設定 → セキュリティ** でネットワークを許可リストに追加するか、`config.php` で `auth.bruteforce.protection.enabled`／IP 許可リストを設定してください。
- **バックグラウンドジョブ（cron）** — 推奨される **Cron** 方式のバックグラウンドジョブを設定し、バージョンのクリーンアップ等の保守処理が確実に実行されるようにしてください。
- **アプリパスワードと二要素認証** — メインアカウントのパスワードは使用しないでください。2FA 有効時はアプリパスワードが必須です。Login Flow v2 が自動で発行します。
- **チェックサム（任意・推奨）** — プラグインは変更検出に Nextcloud の `oc:checksums`（SHA-256）を優先し、無い場合は ETag に自動フォールバックするため設定は不要です。Nextcloud 既定のチェックサム対応を有効のままにしておくと最も正確に検出できます。

## 仕組み（概要）

接続時にプラグインは `/status.php`（メンテナンスモード）と `/ocs/v1.php/cloud/capabilities` を探索し、サーバーのバージョンと利用可能な機能（`checksums`、`files locking` など）を把握します。その後、**デバイスごとの状態データベース**（前回同期成功時点の全ファイルのパス・内容ハッシュ・リモートファイル ID のスナップショット）を維持します。各同期はこのスナップショットとサーバーの `sync-token` を突き合わせ、変更分のみを転送します。Nextcloud 固有の挙動はすべて Capability 検出でゲートされるため、フル機能の Nextcloud Hub でも素の WebDAV サーバーでも同じプラグインが動作します（**Progressive Enhancement**）。

## プライバシーとセキュリティ

- **このプラグインは一切のテレメトリーを収集しません。** 利用データ・解析・クラッシュレポートなどを収集・送信することは一切なく、発生する通信はあなたの Vault とあなた自身の Nextcloud/WebDAV サーバー間の同期のみです。
- アプリパスワード / 認証情報は Obsidian の **credentials ストア**に保存し、`data.json` へ平文保存しません。
- **メインアカウントパスワードは使用・保存しません** — アプリパスワード（手動発行または Login Flow v2）のみ。
- すべての通信は Obsidian の `requestUrl` API を使用。
- Obsidian の設定フォルダ（`.obsidian/`）は既定では同期対象外で、同期されるのはノートやその他の Vault ファイルのみです。**設定フォルダを同期**（下記）で一部を選択的に同期するよう opt-in できます。**コミュニティプラグイン（`.obsidian/plugins/`）とプラグイン自身の同期状態データベースは、設定にかかわらず一切同期されません** — 実行コードや端末固有の状態を含み、端末間で上書きするのは危険なためです。

---

## 制約

- **エンドツーエンド暗号化（E2EE）**は本バージョンの対象外。
- **設定フォルダの同期は opt-in かつ選択的です。** 設定で **設定フォルダを同期** を有効にすると、`.obsidian/` のうち選んだカテゴリ（外観 / テーマ・スニペット / ホットキー / コアプラグイン設定 / ブックマーク）を端末間で同期できます。**コミュニティプラグインとプラグイン自身の同期状態データベースは一切同期されません**（実行コード・端末固有の状態のため）。コアプラグイン設定の同期内容を反映するには、受信側で Obsidian の再起動が必要な場合があります。
- 主に Markdown / テキスト Vault を想定。単一ファイルで数百 MB 級は v1 の設計対象外。
- Vault はローカルストレージに置くこと。他のクラウド同期（iCloud Drive など）と同時に二重管理しない。
- Nextcloud 固有機能には対応サーバーバージョンが必要。古い、または非 Nextcloud のサーバーでは透過的にコア WebDAV 同期へフォールバック。
