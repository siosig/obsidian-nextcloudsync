# Nextcloud Sync for Obsidian

Obsidian Vault と Nextcloud を双方向同期するプラグインです。汎用的な WebDAV ではなく、**Nextcloud 専用**に作られています。

多くの「WebDAV 同期」プラグインはサーバーを単なるファイル置き場として扱い、更新日時を比較してファイルをコピーするだけです。**Nextcloud Sync** は Nextcloud 固有の API（Capabilities、ファイル ID、チェックサム、バージョン、ロック、Login Flow v2）を活用し、同期を *安全* かつ *高速* かつ *簡単* にします。同時に、それらの API が使えない場合は標準 WebDAV へ自動的にフォールバックします。

> English version: see [`README.md`](README.md)

## なぜ Nextcloud 特化なのか（汎用 WebDAV との違い）

| 観点 | 汎用 WebDAV プラグイン | **Nextcloud Sync** |
|------|----------------------|--------------------|
| 変更検出 | 更新日時（時刻ずれに弱く、誤った再アップロードが発生） | **内容ハッシュ** ＋ Nextcloud の `sync-token` / checksums capability による真の差分同期 |
| リネーム / 移動 | 削除＋再作成（履歴を失い、全デバイスで再ダウンロード） | **ファイル ID（`OC-FileId`）追跡** — リネームは全デバイスでリネームのまま、履歴を保持 |
| 削除の安全性 | 完全削除 | **Nextcloud ゴミ箱** 経由 — 復元可能で、不可逆な `DELETE` を行わない |
| セットアップ | アプリパスワードを手動でコピー＆ペースト | **Login Flow v2** — ブラウザで承認するだけで認証情報を自動発行・保存 |
| 大容量ファイル | スキップ、または単一 `PUT` で失敗 | **チャンクアップロード** — 分割・再開可能・チェックサム検証付き |
| 同時編集 | 他者の書き込みを祈るのみ | **Files Locking** — 更新中にサーバーロックを取得し、コンフリクトを*予防* |
| 誤操作からの回復 | なし（手元のコピーが全て） | **サーバーバージョン履歴** — Obsidian 内から過去版を一覧・復元 |
| サーバー停止時 | 不可解なエラー / 中途半端な書き込み | **メンテナンスモード検出**（`/status.php`）と Nextcloud エラーの整形表示 |
| 機能の認識 | なし | **Capabilities 探索**（`/ocs/.../capabilities`）— サーバーが対応する機能だけが有効化される（Progressive Enhancement） |

非 Nextcloud の WebDAV サーバーに接続した場合は、Nextcloud 専用機能を自動的に無効化し、標準的な再帰 WebDAV 同期へフォールバックします。機能は減りますが、そのまま動作します。

## 機能

### コア同期
- Obsidian と Nextcloud の**双方向同期**。
- **ハッシュベース差分同期** — 変更されたファイルのみ転送（全件再スキャンなし）。1,000 ファイル規模の Vault でも数秒で収束。
- **Dry-run プレビュー** — 初回同期の*前*にアップロード / ダウンロード内容を確認し、承認してから実行。
- **アトミック書き込み** — ダウンロード中に転送が中断されても、書きかけや 0 バイトのファイルを Vault に残さない。
- Nextcloud ファイル ID による**リネーム / 移動追跡** — ノートを移動しても全デバイスで再アップロードしない。
- **ゴミ箱経由の削除** — ローカル削除はシステムのゴミ箱へ、リモート削除は Nextcloud のゴミ箱へ（復元可能）。
- **Vault ごとの独立設定** — Vault ごとに別の Nextcloud サーバー / アカウントを設定でき、状態が混在しない。
- **定期自動同期**（間隔は設定可能、`0` で手動のみ）と **Sync Now** コマンド。
- **ファイル変更時の即時同期（ウォッチモード）** — ローカル Markdown を編集すると自動で同期（編集が止まって約2秒後）。設定でトグル ON/OFF、定期同期と併用可。
- **堅牢なリトライ** — 失敗ファイルはスキップ＆キューに積み、次回同期で指数バックオフ付き再試行。Wi-Fi 切断後も自動再開。
- **標準 WebDAV フォールバック** — 任意の WebDAV サーバー（再帰）で動作し、Nextcloud 機能は自動無効化。

### コンフリクト安全性（内容を消さない）
- コンフリクト時に**内容を破棄しない**。
- **インラインコンフリクトマーカー**（Git 形式 `<<<<<<< LOCAL` / `=======` / `>>>>>>> REMOTE`）をファイルに直接書き込み。
- **任意の自動マージ**（`reconcile-text` / diff3）— 異なる領域の編集を統合。既定は OFF。YAML フロントマターは自動マージ対象外。
- ステータスバーの**コンフリクトバッジ** — 未解決件数を表示し、0 で通常表示に戻る（`#conflict` タグ検索と併用可）。

### Nextcloud 強化機能
- **Login Flow v2** — アプリパスワードを手動発行・貼り付けする代わりに、ブラウザ承認だけでセットアップ。認証情報は Obsidian の credentials ストアに保存され、`data.json` に**平文保存しない**。
- **サーバーバージョン履歴** — アクティブなノートについてサーバー保持の全版を新しい順に一覧し、確認のうえ任意の版をアトミックに復元。復元後の内容は無限コンフリクトを起こさずクリーンに同期される。
- **チャンクアップロード** — チャンク閾値を超える大容量ファイル（画像・PDF・音声）を分割・再開可能にアップロード。中断時に不完全なファイルを公開せず、完了時にチェックサム検証。メモリ保護のため別途絶対上限 `maxFileSizeMB` を設定。
- **Files Locking**（*実験的・オプトイン*）— 各更新の直前にファイル単位のサーバーロックを取得し、直後に解放。他クライアント（Nextcloud デスクトップ / Web）との同時書き込みコンフリクトを予防。クラッシュ後の残留ロックも安全に検出・解放。

## 動作要件

- **Obsidian** `1.12.7` 以上（デスクトップ / Electron）。モバイルは現時点で対象外。
- Nextcloud 固有機能には **Nextcloud** Hub 26 「Winter」（`33.0.4`）以上が必要。古いサーバーは接続時に警告し、標準 WebDAV サーバーではコア同期にフォールバック。
- Nextcloud アカウント。認証は **Login Flow v2**（推奨）または手動発行の**アプリパスワード**（メインパスワードは使用不可）。

## インストール

### コミュニティプラグインブラウザから（推奨）
1. Obsidian で **設定 → コミュニティプラグイン** を開く。
2. 制限モードを解除し、**閲覧** をクリックして **Nextcloud Sync** を検索。
3. **インストール** し、**有効化**。

### 手動インストール
1. 最新の [GitHub Release](../../releases) から `main.js` と `manifest.json`（あれば `styles.css`）をダウンロード。
2. `<YourVault>/.obsidian/plugins/obsidian-nextcloudsync/` にコピー。
3. Obsidian をリロードし、**設定 → コミュニティプラグイン** で **Nextcloud Sync** を有効化。

---

## はじめかた

1. **設定 → Nextcloud Sync** を開く。
2. **サーバー URL** を入力（例: `https://cloud.example.com`）。
3. 認証する:
   - **推奨:** **ブラウザでログイン**（Login Flow v2）をクリックし、ブラウザで承認すると認証情報が自動入力・保存される。または
   - **ユーザー名** と手動発行の**アプリパスワード**を入力。
4. （任意）自動同期間隔・**ファイル変更時の即時同期**（ウォッチモード）・自動マージ・チャンク閾値・ロックオプションを調整。
5. **Sync Now** コマンドを実行（または定期同期を待つ）。初回は **Dry-run プレビュー**（`アップロード N 件 / ダウンロード M 件`）が表示されるので承認する。

Vault は Nextcloud 側で Vault 名のフォルダ配下に同期され、複数 Vault がきれいに分離されます。

## 仕組み（概要）

接続時にプラグインは `/status.php`（メンテナンスモード）と `/ocs/v1.php/cloud/capabilities` を探索し、サーバーのバージョンと利用可能な機能（`checksums`、`files locking` など）を把握します。その後、**デバイスごとの状態データベース**（前回同期成功時点の全ファイルのパス・内容ハッシュ・リモートファイル ID のスナップショット）を維持します。各同期はこのスナップショットとサーバーの `sync-token` を突き合わせ、変更分のみを転送します。Nextcloud 固有の挙動はすべて Capability 検出でゲートされるため、フル機能の Nextcloud Hub でも素の WebDAV サーバーでも同じプラグインが動作します（**Progressive Enhancement**）。

---

## プライバシーとセキュリティ

- アプリパスワード / 認証情報は Obsidian の **credentials ストア**に保存し、`data.json` へ平文保存しません。
- **メインアカウントパスワードは使用・保存しません** — アプリパスワード（手動発行または Login Flow v2）のみ。
- すべての通信は Obsidian の `requestUrl` API を使用。
- プラグイン自身のフォルダ（`.obsidian/plugins/obsidian-nextcloudsync/`）は同期対象外。

---

## 制約

- デスクトップ（Electron）のみ。**モバイル**および**エンドツーエンド暗号化（E2EE）**は本バージョンの対象外。
- 主に Markdown / テキスト Vault を想定。単一ファイルで数百 MB 級は v1 の設計対象外。
- Vault はローカルストレージに置くこと。他のクラウド同期（iCloud Drive など）と同時に二重管理しない。
- Nextcloud 固有機能には対応サーバーバージョンが必要。古い、または非 Nextcloud のサーバーでは透過的にコア WebDAV 同期へフォールバック。
