# Nextcloud Sync for Obsidian

**複数のデスクトップ・モバイル端末をまたいで作業しているすべての方へ朗報です。**

あなたが払うコストは、初回インストール時に Vault インデックスの作成完了を待つことだけです。その時点以降は、次が得られます：

- **書いた内容が決して失われない** — マージロジックにより、同じノートを複数端末で同時に編集しても、Vault がクリーンで整合した状態に保たれます。
- **同期が背景へ溶け込む** — Nextcloud の高速な差分同期により、同期にかかる時間をまったく意識しなくなります。
- **Nextcloud 以外でも動作** — 機能はやや減りますが、標準的な WebDAV サーバーに対しても同様に動作します。

---

Obsidian Vault と Nextcloud を双方向同期します。汎用的な WebDAV ではなく、**Nextcloud 専用**に作られています。

多くの「WebDAV 同期」プラグインは、サーバーを単なるファイル置き場として扱います。更新日時を比較し、ファイルをコピーして、うまくいくことを祈るだけです。**Nextcloud Sync** はそうではなく、Nextcloud 自身の API（Capabilities、ファイル ID、チェックサム、バージョン、ロック、Login Flow v2）を活用して、同期を *安全* に・*高速* に・*摩擦なく* します。同時に、それらの API が利用できない場合は素の WebDAV へグレースフルにフォールバックします。

## なぜ Nextcloud 特化なのか（汎用 WebDAV との違い）

| 観点 | 汎用 WebDAV プラグイン | **Nextcloud Sync** |
|------|----------------------|--------------------|
| 変更検出 | 更新日時（時刻ずれに弱く、誤った再アップロードを招く） | **内容ハッシュ** ＋ Nextcloud の `sync-token` / checksums capability による真の差分同期 |
| リネーム / 移動 | 削除＋再作成（履歴を失い、全デバイスで再ダウンロード） | **ファイル ID（`OC-FileId`）追跡** — リネームは全デバイスでリネームのまま、履歴を保持 |
| 削除の安全性 | ハード削除 | **Nextcloud ゴミ箱** 経由 — 復元可能で、不可逆な `DELETE` は決して行わない |
| セットアップ | アプリパスワードを手動でコピー＆ペースト | **Login Flow v2** — ブラウザで承認するだけで認証情報を自動発行・保存 |
| 大容量ファイル | 単一 `PUT` でスキップまたは失敗 | **チャンクアップロード** — 分割・再開可能・チェックサム検証付き |
| 同時編集 | 他者が書き込まないことを祈る | **楽観的同時実行** — すべての更新が `If-Match` プリコンディションを伴うため、他デバイスが変更したリモートはロック往復なしでコンフリクトに変換される（ロストアップデートなし）。サーバー側の **Files Locking** はオプトインで併用可能 |
| 誤操作からの回復 | なし（手元のコピーが全て） | **サーバーバージョン履歴** — Obsidian 内から過去の任意のリビジョンを閲覧・復元 |
| サーバー停止時 | 不可解なエラー／中途半端な書き込み | **メンテナンスモード検出**（`/status.php`）と整形済みの Nextcloud エラーメッセージ |
| 機能の認識 | なし | **Capabilities 探索**（`/ocs/.../capabilities`）— サーバーが対応する機能だけが有効化される（Progressive Enhancement） |

非 Nextcloud の WebDAV サーバーを指定した場合は、Nextcloud 専用機能を自動的に無効化し、標準的な再帰 WebDAV 同期へフォールバックします — 機能は減りますが、そのまま動作します。

## 機能

### コア同期
- Obsidian と Nextcloud の**双方向同期**。
- **ハッシュベースの差分同期** — 変更されたファイルのみを転送（全件再スキャンなし）。1,000 ファイル規模の Vault でも数秒で収束します。
- **アトミック書き込み** — 転送中に中断されたダウンロードが、書きかけや 0 バイトのファイルを Vault に残すことは決してありません。
- Nextcloud ファイル ID による**リネーム / 移動の追跡** — ノートを移動しても全デバイスで再アップロードしません。
- **ゴミ箱経由の削除** — リモート削除は Nextcloud のゴミ箱を使用します（復元可能）。削除をローカル Vault に適用する際は、挙動を一つに固定せず **Obsidian の「削除されたファイル」設定**（システムのゴミ箱／`.trash` へ移動／完全削除）に従います。Vault が追跡するノートの外にあるフォルダやファイル（例: 設定フォルダ配下のファイル）も処理されます。
- **Vault ごとの設定** — 各 Vault が別々の Nextcloud サーバー／アカウントを対象にでき、状態が互いに混ざりません。
- **定期自動同期**（間隔は設定可能、`0` で手動のみ）と **Sync now** コマンド。
- **ファイル変更時の同期（ウォッチモード）** — 任意で、ローカルファイルを編集した直後に即時同期します（入力を止めてから約 2 秒のデバウンス）。設定で ON/OFF を切り替え可能。定期間隔と併用できます。
- **粘り強いリトライ** — 失敗したファイルはスキップしてキューに積み、次回同期で指数バックオフ付きに再試行します。切断された Wi-Fi 接続は自動的に再開されます。
- **標準 WebDAV フォールバック** — 任意の WebDAV サーバーで動作し（再帰）、Nextcloud 機能は自動的に無効化されます。
- **同期ステータスダイアログをステータスで絞り込み** — ステータスダイアログにはチェックボックスの行（Uploaded、Downloaded、Deleted、Merged、Conflicted、Local wins、Remote wins、Error）があり、たとえばコンフリクトだけに集中できます。既定はすべて ON。選択は保存され、Obsidian の再起動を跨いで永続し、すべてのセクションに適用されます。
- **ファイルをリモート版と比較する** *(オプトイン)* — 設定で「リモートと比較（エクスプローラメニュー）」を有効にすると、エクスプローラ内の任意のファイルを右クリックして、ローカルとリモートの更新日時・チェックサム（一致／不一致バッジ付き）・行 diff を比較するポップアップを開けます。その場で **push**（ローカルでリモートを上書き）または **pull**（リモートでローカルを上書き）により差分を解決でき、いずれも確認を挟みます。トグルは再起動不要で即座に反映されます。
- **端末別ログ** — あなたが選んだフォルダ（ファジーなフォルダ選択 UI。既定は Vault ルート）に出力される 2 つのオプトインログ。端末ごとに名前が付くため、複数の端末が互いを上書きすることはありません：
  - **Sync ログ**（`nextcloud-sync_sync_<device>.txt`）— 同期ごとに 1 ブロックを追記。ヘッダにプラグインのバージョンとすべてのマージ関連設定、続いて操作ごとに 1 行でマーカー・パス・ローカル/リモートのチェックサムとサイズを記録します。レベルスイッチで *重要なイベントのみ*（コンフリクト、マージ、side-wins、エラー）か *全操作* かを選べます。
  - **Debug ログ**（`nextcloud-sync_debug_<device>.txt`）— タイムスタンプ付きの診断ログ。詳細度（error / debug / verbose）を選択でき、プラグインのバージョンと全設定のスナップショットを含みます。コンソールのないモバイルでのトラブルシューティングに有用です。終わったら OFF にしてファイルを削除してください。
- **Vault インデックスをリセットする** *(設定 → Maintenance)* — この端末の同期トラッキングインデックスを初回インストール時の状態へ戻し（確認を挟みます）、次回同期で全件を再スキャンさせます。Vault やリモートのファイルは一切削除しません。不整合な同期状態からの復旧に使ってください。

### コンフリクト安全性（内容を決して失わない）
- **自動マージ**（`reconcile-text` / diff3）— 異なる領域の編集を対象とし、両者が重複しない行を変更した場合は YAML フロントマターも含めて統合します（既定 ON）。
- **拡張子によるマージ範囲の限定** — 設定可能な拡張子（既定 `md`、`txt`）のファイルのみがテキストマージされます。それ以外のファイル（画像、PDF、バイナリ）は決してマージされず、マーカーが書き込まれることもありません。
- クリーンにマージできないものに対する**コンフリクト解決ポリシー**: **エラー**（両側を一切触らず、報告して次回同期で再試行 — 既定）、**ローカル優先**（ローカルでリモートを上書き）、**リモート優先**（リモートでローカルを上書き）、**コンフリクトマーカー**（Git スタイルの `<<<<<<< LOCAL` / `=======` / `>>>>>>> REMOTE` をファイルに書き込み。テキストファイルのみ — それ以外のファイルはエラーにフォールバック）。
- ステータスバーの**コンフリクトバッジ** — 未解決コンフリクトの件数を表示します（0 で通常表示に戻ります。`#conflict` タグ検索との相性が良いです）。

### Nextcloud 強化機能
- **Login Flow v2** — アプリパスワードを手動で発行・貼り付けする代わりに、ブラウザでの承認でセットアップします。認証情報は Obsidian の secret credentials ストアに保存され、`data.json` に**平文で書かれることはありません**。
- **サーバーバージョン履歴** — アクティブなノートについて、サーバーが保持するすべてのリビジョンを新しい順に一覧し、確認のうえ任意のものをアトミックに復元します。復元された内容は、無限のコンフリクトループを引き起こすことなくクリーンに同期されます。
- **チャンクアップロード** — チャンク閾値を超える大きな添付（画像、PDF、音声）は分割され、再開可能にアップロードされます。中断されたアップロードが不完全なファイルを公開することはなく、完了時にチェックサムで検証されます。メモリを守るための別個の絶対上限 `maxFileSizeMB` も備えています。
- **Files Locking** *(実験的、既定 OFF)* — 任意で、各更新の直前にファイル単位のサーバーロックを取得し、直後に解放します。**既定は OFF：** ロストアップデートに対する安全性は、代わりに常時有効の `If-Match` プリコンディションが提供します（他クライアントが変更したリモートは 412 を返し、エンジンがそれをコンフリクトに変換します）。これにより、ファイルごとの LOCK/UNLOCK 往復を回避します。二重の安全策が欲しい場合に有効化してください。Nextcloud の files-locking アプリが必要で、アプリが無い場合は無効のままです。

## モバイル（iOS / Android）

モバイルに対応しており、プラットフォームを考慮したいくつかの差分があります（デスクトップの挙動は変わりません）：

- **モバイルでは自動同期が既定で OFF です。** OS がバックグラウンドのタイマーを停止するため、定期自動同期と「ファイル変更時の同期」は無効化されます（グレーアウト）。**Sync now** を使うか、**起動時に同期**（モバイルでは既定 OFF）を有効にして、アプリ起動の数秒後に一度同期してください。
- **起動時に同期** は両プラットフォームの新しい設定です（デスクトップ: ON、1 秒／モバイル: OFF）。
- **モバイルでは大容量ファイルがスキップされます** — 「最大ファイルサイズ」上限（`0` で無制限）を超えるものは、メモリ不足によるクラッシュを避けるためスキップされ、スキップは報告されます。
- **モバイルには進捗 UI がありません** — エラー通知のみが表示されます。
- **ネットワーク並列数** は設定可能で、初回の既定値は端末の利用可能メモリから導出されます — デスクトップとモバイルで一律、プラットフォーム固有の分岐はありません: RAM 8 GB 以上で **16**、4 GB 以上で **8**、それ未満で **4**、端末がメモリ量を報告しない場合（モバイルに多い）は **3**。転送は上限付きの並列度で実行され — この数と、同時にやり取り中のバイト総量の予算（モバイルでは小さめ）の両方で上限化され、大きなファイルがメモリを使い切ることはありません — また、同一フォルダへのアップロードは直列化され、サーバー側のロック競合を回避します。
- **Wi-Fi のみで同期** はセルラー時にスキップします（Android／デスクトップ）。**iOS では利用不可**（ネットワーク種別の API が無い）で、トグルは無効化されます。
- **モバイルでは Sync now が結果通知を表示します**（アップロード／ダウンロード／コンフリクト、または「すでに最新です」）— ステータスバーが無いためです。サインインしていない状態でタップしても無効のままで、設定画面には明確な「まだサインインしていません」バナーが表示されます。
- デバッグモード（診断ログ）はモバイルでも利用可能で、同期の挙動は変えません。

## 動作要件

- **Obsidian** `1.11.4` 以降（本プラグインは `1.11.4` で導入された secret-storage API を使用します）。デスクトップ（Electron）とモバイル（iOS / Android）に対応します。
- Nextcloud 固有機能には **Nextcloud** Hub 26 「Winter」（サーバー `33`）以降を**推奨**します。これより古い Nextcloud サーバーはもはやブロックされません — 接続・同期は可能ですが、設定画面に推奨バナーが表示され、一部の機能が制限される場合があります。素の WebDAV サーバーはコア同期にフォールバックします。
- Nextcloud アカウント。**Login Flow v2**（推奨）または手動発行の**アプリパスワード**（メインパスワードは決して使わない）で認証できます。

## インストール

### コミュニティプラグインブラウザから（推奨）
1. Obsidian で **設定 → コミュニティプラグイン** を開きます。
2. 制限モードを解除し、**閲覧** をクリックして **Nextcloud Sync** を検索します。
3. **インストール** し、続けて **有効化** します。

### 手動インストール
1. 最新の [GitHub Release](../../releases) から `main.js` と `manifest.json`（あれば `styles.css`）をダウンロードします。
2. それらを `<YourVault>/.obsidian/plugins/nextcloud-sync/` にコピーします。
3. Obsidian をリロードし、**設定 → コミュニティプラグイン** で **Nextcloud Sync** を有効化します。

---

## はじめかた

1. **設定 → Nextcloud Sync** を開きます。
2. **サーバー URL** を入力します（例: `https://cloud.example.com`）。
3. 認証します：
   - **推奨:** **ブラウザでログイン**（Login Flow v2）をクリックし、ブラウザで承認すると、認証情報が自動入力・保存されます。**または**
   - **ユーザー名** と手動発行の**アプリパスワード**を入力します。
4. （任意）自動同期間隔、**ファイル変更時の同期**（ウォッチモード）、自動マージ、チャンク閾値、ロックオプションを調整します。
5. **Sync now** コマンドを実行します（または定期同期を待ちます）。初回実行では Vault とリモートの全件スキャンを行い、必要なものを転送します。以降の同期は差分的になります。

あなたの Vault は Nextcloud 側で Vault 名のフォルダへ同期され、複数の Vault がきれいに分離された状態に保たれます。

## Nextcloud サーバー側機能の有効化

2 つの強化機能はサーバー側の Nextcloud アプリに依存します。いずれも **Nextcloud 管理者が一度だけ有効化**すれば済みます。プラグインは capabilities API を通じてそれらを検出します — アプリが無ければ、該当機能は単に無効のままになります（エラーにはなりません）。

### File Locking

ロックは Nextcloud の **Temporary files lock** アプリ（アプリ ID `files_lock`、Nextcloud 24 以降で利用可能。v34 以降は Nextcloud Hub に同梱）を使用します。

- **Web UI（管理者）:** 管理者としてサインイン → 右上のプロフィールメニュー → **Apps** → **Temporary files lock** を検索 → **Enable**（未インストールなら先にダウンロード）。
- **コマンドライン（`occ`）** — Web サーバーユーザー（多くは `www-data`）として実行します：
  ```bash
  sudo -u www-data php /var/www/nextcloud/occ app:enable files_lock
  ```
- その後、プラグイン設定で **File Locking (Experimental)** を有効にして再同期してください。（Docker: `docker exec -u www-data <container> php occ app:enable files_lock`）

### バージョン履歴

サーバー側のバージョンは組み込みの **Versions** アプリ（アプリ ID `files_versions`）が提供します。標準的な Nextcloud インストールでは**既定で有効**のため、通常は何もする必要がありません。

- 無効化されている場合は再有効化します: **Apps → Versions → Enable**、または：
  ```bash
  sudo -u www-data php /var/www/nextcloud/occ app:enable files_versions
  ```
- バージョンはファイルが変化するたびに自動的に作成されます。Obsidian の **Show version history** コマンドで閲覧・復元できます。
- 過去のリビジョンが存在するには、サーバー上でそのノートが少なくとも一度は変更されている必要があります。保持世代は管理者がサーバー側で設定します（`config.php` の `versions_retention_obligation`）。

### その他、確認しておく価値のある設定

これらは厳密には必須ではありませんが、セルフホスト環境ではスムーズで信頼できる同期のためにしばしば注意が必要になります。いずれもサーバー側（管理者）の設定です。

- **信頼済みドメイン（Trusted domains）** — 接続先のホストが `config.php` の `trusted_domains` に列挙されている必要があります。さもないとサーバーがリクエストを拒否します。必要に応じてドメイン／IP を追加してください。
- **HTTPS とリバースプロキシ（Login Flow v2 で重要）** — リバースプロキシの背後では、`overwriteprotocol => 'https'`、`overwritehost`、`overwrite.cli.url`、`trusted_proxies` を正しく設定してください。これらが誤っていると、Login Flow v2 が返す URL（およびダウンロード）が誤ったスキーム／ホストを指して失敗することがあります。プラグインのサーバー URL は常に `https://` を使ってください。
- **アップロードサイズ上限（チャンクアップロード／大容量添付向け）** — PHP の `upload_max_filesize` と `post_max_size`、および Web サーバーのボディ上限（nginx の `client_max_body_size`、例: `0` か大きい値）を引き上げてください。チャンクアップロードは小さなチャンクを送りますが、最終的な結合や非常に大きなファイルは依然としてこれらの上限に達します。
- **リクエストタイムアウト** — 大きな Vault や大きいファイルでは、PHP の `max_execution_time` や php-fpm／Web サーバーのタイムアウト（例: nginx の `fastcgi_read_timeout`）を増やしてください。プラグイン自身のネットワークタイムアウトは設定で調整できます。
- **ブルートフォース保護** — Nextcloud は単一 IP からの繰り返しリクエストをスロットリングし、特に複数のデバイスが同じネットワークから同期する場合や認証エラーの後に HTTP 429 を返すことがあります。これに遭遇したら、**管理設定 → セキュリティ** でそのネットワークを許可リストに追加するか、`config.php` で `auth.bruteforce.protection.enabled`／IP 許可リストを設定してください。
- **バックグラウンドジョブ（cron）** — Nextcloud が推奨する **Cron** 方式のバックグラウンドジョブを設定し、バージョンのクリーンアップやその他の保守処理が確実に動くようにしてください。
- **アプリパスワードと二要素認証** — メインアカウントのパスワードは決して使わないでください。2FA が有効な場合はアプリパスワードが必須です。Login Flow v2 が自動で発行します。
- **チェックサム（任意・推奨）** — プラグインは変更検出に Nextcloud の `oc:checksums`（SHA-256）を優先し、無い場合は自動的に ETag へフォールバックするため、設定は不要です。Nextcloud の既定のチェックサム対応を有効のままにしておくと、最も正確な検出が得られます。

## 仕組み（概要）

接続時、プラグインは `/status.php`（メンテナンスモード）と `/ocs/v1.php/cloud/capabilities` を探索し、サーバーのバージョンと、どの機能（`checksums`、`files locking`、…）が利用可能かを把握します。その後、**端末ごとの状態データベース** — 最後に成功した同期時点での全ファイルのパス・内容ハッシュ・リモートファイル ID のスナップショット — を維持します。各同期はその時点の状態をこのスナップショットおよびサーバーの `sync-token` と差分比較し、変更分のみを転送します。Nextcloud 固有のあらゆる挙動は capability 検出の背後にゲートされるため、同じプラグインがフル機能の Nextcloud Hub でも素の WebDAV サーバーでも同様に動作します（**Progressive Enhancement**）。

---

## テストと信頼性

同期の正しさは広範な自動テストで担保しています：**数百件の高速な純ロジックテスト**（変更のたびに実行）に加え、**実 Nextcloud サーバーに対して 2 端末を実際に動かす E2E テスト群**（コンフリクト解決のオプション全組合せマトリクスやマルチデバイス収束を含む）を備えています。

これらのテストは、同期不整合状態 — **データ消失、無限に続く再アップロード／再ダウンロード、リモートの変更がローカルに反映されない、ローカルの変更がリモートに同期されない** — を防ぐために用意しています。とはいえ、あらゆるケースを完全に網羅できるとは限らず、意図しない動作の可能性を完全に否定することはできません。**そのような現象を見つけたら、ぜひ気軽に [issue](https://github.com/siosig/obsidian-nextcloudsync/issues) に投げてください。できる限り迅速に対応します。**

## プライバシーとセキュリティ

- **このプラグインはテレメトリーを一切収集しません。** 利用データ・解析・クラッシュレポートはどこにも収集・送信されません。発生する唯一のネットワーク通信は、あなたの Vault とあなた自身の Nextcloud/WebDAV サーバーとの間の同期だけです。
- アプリパスワード／認証情報は Obsidian の **secret credentials ストア**に保管され、`data.json` に平文で書かれることはありません。
- **あなたのメインアカウントのパスワードは使用も保存もされません** — 使われるのはアプリパスワード（手動発行または Login Flow v2 経由）のみです。
- すべてのネットワーク通信は Obsidian の `requestUrl` API を使用します。
- Obsidian の設定フォルダ（`.obsidian/`）は既定で同期から除外されます — 同期されるのはあなたのノートとその他の Vault ファイルのみです。**設定フォルダを同期**（下記参照）で、その一部を選択的に同期するようオプトインできます。**コミュニティプラグイン（`.obsidian/plugins/`）とプラグイン自身の同期状態データベースは、設定にかかわらず決して同期されません** — これらは実行可能コードと端末固有の状態を保持しており、端末間で上書きするのは安全ではないためです。

---

## 制約

- **エンドツーエンド暗号化（E2EE）** は本バージョンの対象外です。
- **設定フォルダの同期はオプトインかつ選択的です。** 設定で **設定フォルダを同期** を有効にすると、`.obsidian/` のうち選んだカテゴリ（外観、テーマとスニペット、ホットキー、コアプラグイン設定、ブックマーク）を端末間で同期できます。**コミュニティプラグインとプラグイン自身の同期状態データベースは決して同期されません**（実行可能コード／端末固有の状態のため）。コアプラグイン設定の同期された変更を反映させるには、もう一方の端末で Obsidian の再起動が必要な場合があります。
- 主に Markdown／テキストの Vault 向けに設計されています。単一ファイルで数百 MB 級のものは v1 の設計目標を超えます。
- Vault はローカルストレージに置いてください — 別のクラウド同期（例: iCloud Drive）と同時に二重管理しないでください。
- Nextcloud 固有の機能には互換性のあるサーバーバージョンが必要です。古いサーバーや非 Nextcloud のサーバーでは、透過的にコア WebDAV 同期へフォールバックします。
